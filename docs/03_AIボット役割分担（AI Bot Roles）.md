## 🕐 ① 利用時間「7:30〜18:00」に制限するのはアリか？

👉 **大いにアリ（むしろベストプラクティス）**です。

### なぜ良いか（現実運用の観点）

* ✅ 情シス対応時間＝勤務時間と一致
* ✅ 夜間の"誤回答・暴走"リスクを防げる
* ✅ インシデントは人間にエスカレーションできる
* ✅ 監査・説明責任が取りやすい

つまりこれは
**「AIを便利に使う」ではなく「AIを安全に運用する」設計**です。

### 実装イメージ（超シンプル）

```
現在時刻をチェック
  ├─ 7:30〜18:00 → BOT通常対応
  └─ それ以外   → 定型応答＋案内のみ
```

#### 時間外のBOT応答例（事故らない）

* 「現在は自動受付のみです」
* 「緊急時は〇〇へ連絡してください」
* 「内容は記録され、営業時間内に確認されます」

👉 **時間外は"賢くしない"のが正解**
（ここ、重要です）

## ② 調査・要約補助として Perplexity はどうか？

👉 **役割限定なら「非常に優秀」**です。

### Perplexityの評価（データクレンジング視点）

**強み**

* 🔍 出典付き調査が得意
* 🧾 最新情報に強い
* 📚 複数情報源の要約が速い
* 🧠 「なぜそう言えるか」を示せる

**弱み**

* ❌ 判断・責任ある最終回答は不向き
* ❌ 社内事情（ローカルルール）を知らない
* ❌ 情シス文脈での"言い切り"は危険

👉 結論：
**「調査員」としては◎、
「窓口」や「最終回答者」には×**

👉 **Perplexityは「裏方」専用**が最適解。
